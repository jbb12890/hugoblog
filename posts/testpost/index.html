これをかくとtocがきく#

開発用GitHubレポジトリ（private）と、デプロイ用GitHubレポジトリ（public）で運用する。

GitHub Actionsでエラーが発生。

  1. 開発側を管理するGitHubアカウントでの秘密鍵設定でミス
    • コメント部分を貼り付けたらうまくいった。（—–BEGIN RSA PRIVATE KEY—–）
  2. external_repository: の値
    • username/external-repositoryの末尾に.github.ioをつけていた。

認証部分でのエラーを1.で解消したあとに、別のエラーがでて2.で解決したのでメモとして残しておく。

1.は初めてSSH鍵の作成をやったのでいい勉強になった。

2.は別のブログで.github.ioありで記載がありハマった。。公式マニュアルもFor example:のとおり記載すればよかったが、Use case:の部分にexternal_repository: peaceiris/peaceiris.github.ioの記載がありハマってしまった。

こうしたちょっとのミスでうまく行かなくてハマってしまうことがよくあるが、解決するとすごくスッキリする。
